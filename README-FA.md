## 🚀 شروع سریع

**پیش‌نیازها:**
- Ubuntu 20.04+ یا Debian 11+
- حداقل 1GB رم (Conduit بسیار سبک است!)
- دامنه داخلی برای فدراسیون، یا آدرس IP فقط برای حالت ایزوله/تست
- پورت‌ها: 80، 443 (یا سفارشی)، 3478، 5349 (UDP)

**نصب با یک دستور:**

```bash
git clone https://github.com/MatinSenPai/zanjir.git ~/zanjir
cd ~/zanjir
sudo bash install.sh
```

**دسترسی:**
- **اپلیکیشن وب**: `https://your-domain.com`
- **پنل مدیریت**: `https://your-domain.com/admin`

## 📦 نصب

### مرحله 1: کلون کردن مخزن

```bash
git clone https://github.com/MatinSenPai/zanjir.git
cd zanjir
```

### مرحله 2: اجرای نصب‌کننده

```bash
sudo bash install.sh
```

**سوالات نصب:**

1. **آدرس سرور** - دامنه داخلی یا IP (مثلاً `matrix-a.example.internal` یا `192.168.1.50`)
2. **حالت استقرار** - `isolated` یا `federated`
3. **حالت گواهی** - `public` یا `private` برای نصب‌های دامنه‌ای
4. **ایمیل مدیریت** - فقط وقتی CA عمومی انتخاب شده باشد
5. **مسیر فایل گواهی/کلید** - فقط وقتی CA خصوصی/داخلی انتخاب شده باشد
6. **پورت HTTPS** - پیش‌فرض: 443 (Enter)، یا سفارشی (مثلاً `8443`)

### حالت‌های پشتیبانی‌شده

- `isolated + IP`: گواهی داخلی Caddy، فدراسیون غیرفعال، فقط برای استقرار ایزوله یا تست
- `isolated + domain + public CA`: استقرار تک‌سروری با TLS عمومی
- `isolated + domain + private CA`: استقرار تک‌سروری با فایل PEM اپراتور
- `federated + domain + public CA`: گزینه ترجیحی وقتی اعتبارسنجی CA عمومی واقعاً ممکن باشد
- `federated + domain + private CA`: fallback اصلی برای شبکه‌های داخلی

برای فدراسیون کامل پشتیبانی نمی‌شود:

- HTTP ساده بین homeserverها
- حالت IP به‌عنوان مسیر عادی فدراسیون

### مرحله 3: ثبت‌نام کاربر

بعد از اتمام نصب:

1. **مرورگر خود را باز کنید** و به سرور خود بروید:
   ```
   https://your-domain.com
   ```

2. روی **"ساخت حساب کاربری"** کلیک کنید

3. از طریق رابط وب ثبت‌نام کنید

از شما درخواست می‌شود نام کاربری و رمز عبور تنظیم کنید.

نکته فعلی:

- مسیر قدیمی ساخت حساب Dendrite دیگر استفاده نمی‌شود
- نصب فدره اکنون `FEDERATION_ENABLED=true` را در `.env` می‌نویسد
- در حالت CA خصوصی/داخلی فایل‌های PEM در `./certs` قرار می‌گیرند

## 🎛️ مدیریت سرور

بعد از نصب، از دستور `zanjir` برای مدیریت سرور خود استفاده کنید:

```bash
zanjir
```

### دستورات موجود

```bash
zanjir              # منوی تعاملی
zanjir status       # نمایش وضعیت سرویس‌ها
zanjir logs         # مشاهده لاگ‌ها (تمام سرویس‌ها)
zanjir logs conduit # مشاهده لاگ سرویس خاص
zanjir restart      # ری‌استارت تمام سرویس‌ها
zanjir start        # شروع تمام سرویس‌ها
zanjir stop         # توقف تمام سرویس‌ها
zanjir update       # آپدیت به آخرین نسخه
zanjir backup       # پشتیبان‌گیری از تمام داده‌ها
zanjir uninstall    # حذف کامل زنجیر
```

## 🛡️ پنل مدیریت

### دسترسی

به آدرس: `https://your-domain.com/admin` بروید

با اطلاعات حساب خود وارد شوید.

محدودیت فعلی:

- این پنل هنوز یک رابط کمکی سبک با منطق مجوزدهی ضعیف در `admin/app.py` است
- فعلاً آن را مرز امنیتی قطعی برای عملیات مدیریتی در نظر نگیرید

### ویژگی‌ها

#### داشبورد
- تعداد کل کاربران
- کاربران فعال
- کل اتاق‌ها

#### مدیریت کاربران
- مشاهده همه کاربران
- غیرفعال کردن حساب‌های کاربری
- حذف کاربران
- مشاهده وضعیت کاربر

#### لاگ فعالیت
- ردیابی تمام اقدامات مدیران
- زمان و آدرس IP
- ردیابی کاربر هدف
- لاگ‌های دقیق اقدام

### اقدامات مدیریتی

**غیرفعال کردن کاربر:**
1. به صفحه `کاربران` بروید
2. کاربر را پیدا کنید
3. روی **"غیرفعال"** کلیک کنید

**حذف کاربر:**
1. به صفحه `کاربران` بروید
2. کاربر را پیدا کنید
3. روی **"حذف"** کلیک کنید → تایید کنید

تمام اقدامات به‌طور خودکار لاگ می‌شوند و در صفحه **لاگ‌ها** قابل مشاهده هستند.

## 📞 تماس صوتی/تصویری

زنجیر شامل یک سرور TURN (coturn) برای تماس‌های صوتی/تصویری قابل اعتماد است.

### نحوه کار

سرور TURN به کاربران پشت NAT/فایروال کمک می‌کند متصل شوند:
- P2P مستقیم در صورت امکان
- رله TURN در صورت نیاز
- بازگشت خودکار

### پیکربندی فایروال

**پورت‌های مورد نیاز:**

| پورت | پروتکل | هدف |
|------|---------|------|
| 3478 | UDP | STUN/TURN |
| 5349 | UDP | TURN-TLS |

**مثال UFW:**

```bash
sudo ufw allow 3478/udp
sudo ufw allow 5349/udp
```

### تست تماس‌ها

1. دو حساب بسازید
2. یک اتاق بسازید، هر دو کاربر را دعوت کنید
3. روی آیکون تلفن/ویدیو کلیک کنید
4. تماس را در سمت دیگر قبول کنید

## 🔧 پورت سفارشی

اگر پورت 443 قبلاً در حال استفاده است، می‌توانید از پورت سفارشی استفاده کنید.

### نصب با پورت سفارشی

در حین اجرای `install.sh`، پورت دلخواه خود را وارد کنید:

```
HTTPS port (default: 443): 8443
```

این کار:
- از پورت 8443 برای HTTPS استفاده می‌کند
- از پورت 8080 برای HTTP استفاده می‌کند (محاسبه خودکار)
- تمام پیکربندی‌ها را به‌روز می‌کند

### دسترسی با پورت سفارشی

```
https://your-domain.com:8443
```

### تغییر پورت بعد از نصب

1. فایل `.env` را ویرایش کنید:
   ```bash
   nano .env
   ```
2. `HTTPS_PORT` و `HTTP_PORT` را تغییر دهید
3. سرویس‌ها را ری‌استارت کنید:
   ```bash
   docker compose down
   docker compose up -d
   ```

## 🔍 رفع مشکلات

### مشکلات رایج

#### نصب Docker شکست می‌خورد (VPS ایران)

**خطا:** `connection refused` به `download.docker.com`

**راه‌حل:** اسکریپت به‌طور خودکار از میرورهای ایرانی استفاده می‌کند:
- `docker.arvancloud.ir`
- `registry.docker.ir`

#### ثبت‌نام کار نمی‌کند

**بررسی کنید:**
1. در docker-compose.yml بررسی کنید: `CONDUIT_ALLOW_REGISTRATION: "true"`
2. `element-config.json` را بررسی کنید: `"UIFeature.registration": true`
3. کانتینرها را ری‌استارت کنید:
   ```bash
   docker compose restart
   ```

#### تماس صوتی شکست می‌خورد

**راه‌حل‌ها:**
1. سرور TURN در حال اجرا است بررسی کنید:
   ```bash
   docker ps | grep coturn
   ```
2. فایروال اجازه UDP 3478، 5349 را می‌دهد بررسی کنید
3. سکرت TURN در `.env` با متغیر محیطی `TURN_SECRET` مطابقت دارد بررسی کنید

#### ورود به پنل مدیریت شکست می‌خورد

**راه‌حل‌ها:**
1. سلامت سرویس‌ها و دسترسی homeserver را بررسی کنید:
   ```bash
   zanjir status
   ```
2. لاگ کانتینر ادمین را بررسی کنید:
   ```bash
   docker logs zanjir-admin
   ```
3. فعلاً پنل مدیریت را آزمایشی فرض کنید؛ بررسی مستقیم سرویس‌ها و compose قابل اتکاتر است.

#### فدراسیون بین سرورها کار نمی‌کند

**بررسی کنید:**
1. هر دو سرور با حالت `federated` نصب شده باشند
2. هر دو سرور از دامنه داخلی پایدار استفاده کنند، نه IP خام
3. هر دو سمت همان زنجیره CA خصوصی/داخلی را trust کرده باشند یا گواهی عمومی معتبر داشته باشند
4. خروجی `/.well-known/matrix/server` درست باشد:
   ```bash
   curl -k https://your-domain/.well-known/matrix/server
   ```
5. در `docker compose logs caddy conduit` خطای گواهی دیده نشود

#### پورت قبلاً در حال استفاده است

**راه‌حل:** در حین نصب از پورت سفارشی استفاده کنید، یا پورت را در `.env` تغییر دهید

## ❓ سوالات متداول

### عمومی

**س: آیا می‌توانم از آدرس IP به‌جای دامنه استفاده کنم؟**  
ج: بله، اما فقط برای استقرار ایزوله یا تست. حالت IP از CA داخلی Caddy استفاده می‌کند و مسیر پشتیبانی‌شده برای فدراسیون کامل نیست.

**س: آیا فدراسیون فعال است؟**  
ج: فقط وقتی در نصب‌کننده حالت `federated` را انتخاب کنید. حالت `isolated` فدراسیون را غیرفعال نگه می‌دارد.

**س: آیا کاربران می‌توانند به‌صورت خارجی تماس ویدیویی داشته باشند؟**  
ج: تماس‌ها برای کاربرانی که به همان استقرار و TURN دسترسی دارند کار می‌کند. رفتار بین‌سروری به تنظیم درست فدراسیون و گواهی هر سرور بستگی دارد.

### امنیت

**س: رمزهای عبور چگونه ذخیره می‌شوند؟**  
ج: هش شده با Argon2 در RocksDB.

**س: آیا رمزنگاری سرتاسر پشتیبانی می‌شود؟**  
ج: بله! Element/Matrix به‌طور پیش‌فرض از E2EE پشتیبانی می‌کند.

**س: لاگ فعالیت چطور؟**  
ج: اقدامات مدیر در SQLite لاگ می‌شوند (`admin/audit_log.db`).

### عملکرد

**س: چند کاربر را می‌تواند مدیریت کند؟**  
ج: Conduit بسیار سبک است. یک VPS 1GB می‌تواند ~100-500 کاربر را با راحتی مدیریت کند. Conduit ~50MB رم استفاده می‌کند در مقابل 200-500MB Dendrite.

**س: پشتیبان‌گیری چطور؟**  
ج: از volumeهای فعال Docker پشتیبان بگیرید، به‌ویژه `zanjir-conduit-data`، `zanjir-caddy-data` و `zanjir-admin-data`. فرمان `zanjir backup` کمک فعلی پروژه است.
